## Building Documentation

### HTML Documentation

```bash
cd docs
pip install sphinx sphinx-rtd-theme
make html
```

Output will be in `docs/_build/html/index.html`

### Publishing to Read the Docs

The documentation is now ready to be published on Read the Docs:

1. **Connect Repository**: Link GitHub repo to readthedocs.org
2. **Configure**: RTD will automatically detect docs/ and conf.py
3. **Build**: Automatic builds on every commit
4. **Access**: Documentation will be available at catalogmx.readthedocs.io

## Modern Python Package Structure

### pyproject.toml Benefits

**Before** (old setup.py):
- Multiple files (setup.py, setup.cfg, MANIFEST.in)
- Inconsistent configuration
- Hard to maintain

**After** (pyproject.toml):
- Single source of truth (PEP 621)
- Modern standard (Python 3.10+)
- Tool integration (black, ruff, mypy, pytest)
- Easy to maintain

### Installation Modes

```bash
# Basic installation
pip install catalogmx

# Development mode
pip install -e ".[dev]"

# With documentation tools
pip install -e ".[docs]"

# Everything
pip install -e ".[all]"
```
